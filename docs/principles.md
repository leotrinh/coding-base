# Engineering Principles

This repository is built around these review principles:

- 4-Eyes Principle
- SOLID
- DRY
- YAGNI
- KISS

## 1. 4-Eyes Principle
Act as the second pair of eyes.
Ask:
- Is the structure understandable without tribal knowledge?
- Can another developer trace the main flow quickly?
- Are logical mistakes, security gaps, or edge cases visible early?

## 2. SOLID
- Single Responsibility
- Open/Closed
- Liskov Substitution
- Interface Segregation
- Dependency Inversion

## 3. DRY
Do not repeat stable logic unnecessarily.  
But do not create shared abstractions too early.

## 4. YAGNI
Do not build for an imaginary roadmap.

## 5. KISS
Simple beats clever.
